# 0002 — Цены заказа вычисляются на фронтенде и доверяются бэкендом

- **Status:** Accepted
- **Confidence:** HIGH — `frontend/src/screens/PlaceOrderScreen.js` вычисляет `itemsPrice`, `taxPrice`, `shippingPrice`, `totalPrice` и отправляет их в теле POST-запроса; `backend/controllers/orderController.js::addOrderItems` сохраняет полученные значения в MongoDB без перепроверки

## Context

При оформлении заказа нужно показать покупателю итоговую стоимость (товары + доставка + налог) и зафиксировать её в базе. Возникает вопрос: где считать — на клиенте или на сервере?

## Decision

Все ценовые расчёты выполняются в компоненте `PlaceOrderScreen.js`:

```
itemsPrice   = сумма (price × qty) по всем товарам в корзине
shippingPrice = itemsPrice > 100 ? 0 : 100
taxPrice      = 0.15 × itemsPrice
totalPrice    = itemsPrice + shippingPrice + taxPrice
```

Готовые числа передаются в `POST /api/orders`. Контроллер `addOrderItems` принимает их из `req.body` и сохраняет в `Order` документ напрямую. Сервер не делает независимого расчёта и не сверяет полученные значения с актуальными ценами из БД.

Цены товаров для расчёта берутся из Redux store (`cart.cartItems`), куда они попали в момент `addToCart` через `GET /api/products/:id`.

## Alternatives Considered

- **Server-side calculation** — backend получает список `{ productId, qty }`, сам тянет цены из MongoDB, считает итог. Исключает возможность подмены цены клиентом. Сложнее: нужен дополнительный DB-запрос и логика расчёта на бэкенде.
- **Двойная проверка** — клиент считает для отображения, сервер пересчитывает и сверяет перед сохранением; расхождение → 400. Максимальная защита, максимальная сложность.

## Consequences

**Плюсы:**
- Мгновенный отклик UI — пересчёт без round-trip на сервер
- Меньше кода на бэкенде
- Легко показывать промежуточные цифры на каждом шаге чекаута

**Минусы:**
- **Уязвимость к подмене цены:** клиент может отправить произвольный `totalPrice`, сервер сохранит его и передаст в PayPal. При текущей интеграции PayPal получает сумму напрямую из `order.totalPrice`, не из независимого источника
- Цена в заказе фиксируется в момент `addToCart`, а не в момент оформления — если цена товара изменилась между этими событиями, покупатель платит старую цену
- Бизнес-правила (ставка налога 15%, порог доставки 100) живут в компоненте, а не в конфиге или на сервере
